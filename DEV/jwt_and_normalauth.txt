MongoDB Authentication

This is between:

Backend  <------>  MongoDB

MongoDB asks:

"Is this backend allowed to access me?"

The backend uses its database username and password (or connection string).

Example:

mongoose.connect(process.env.MONGO_URI);

Once connected, the backend can query the database.

JWT Authentication

JWT is between:

User  <------>  Backend

The backend asks:

"Is this user logged in?"

JWT has nothing to do with MongoDB authentication.

Complete Flow

Imagine you have a login page.

Step 1: User enters credentials
Email: aryan@gmail.com
Password: mypassword

The browser sends:

POST /login

to your backend.

Step 2: Backend checks the database

The backend queries MongoDB.

const user = await User.findOne({
    email: "aryan@gmail.com"
});

MongoDB simply returns the stored user document.

Example:

{
    "_id": "123",
    "email": "aryan@gmail.com",
    "password": "$2b$10$..."
}
Step 3: Password verification

The backend compares the entered password with the stored hashed password using a library like bcrypt.

const match = await bcrypt.compare(password, user.password);

If the passwords don't match:

Login Failed
Step 4: Backend creates a JWT

If the password is correct:

const token = jwt.sign(
    {
        userId: user._id,
        email: user.email
    },
    SECRET_KEY,
    { expiresIn: "1h" }
);

A JWT consists of three parts:

Header.Payload.Signature

Example:

xxxxx.yyyyy.zzzzz

The payload might contain:

{
    "userId": "123",
    "email": "aryan@gmail.com"
}

The signature is created using your secret key so that no one can tamper with the token.

Step 5: Backend sends the JWT
{
   "token":"eyJhbGciOi..."
}

The frontend stores it (commonly in an HTTP-only cookie or sometimes local storage).

Step 6: User requests another page

Suppose the user wants their profile.

GET /profile

The frontend sends:

Authorization: Bearer eyJhbGc...
Step 7: Backend verifies the JWT
jwt.verify(token, SECRET_KEY);

If the signature is valid and the token hasn't expired:

User is authenticated.

Otherwise:

401 Unauthorized
Why use JWT?

Imagine there are no JWTs.

Without JWT

Every request would require:

Email
Password
Request 1
Email
Password

Request 2
Email
Password

Request 3
Email
Password

This is inconvenient and insecure.

With JWT

The user logs in once.

Email
Password

The backend returns:

JWT

Now every request only sends:

JWT

The backend verifies the token instead of asking for the password again.

Where does MongoDB fit?
               Login
User ----------------------> Backend
                               |
                               | Query user
                               v
                           MongoDB

                    Password correct?
                               |
                               v
                        Generate JWT
                               |
                               v
User <---------------------- JWT

Later:

User
   |
   | JWT
   v
Backend
   |
   | Verify JWT
   |
   | Query MongoDB if needed
   v
MongoDB

Notice that MongoDB never sees the JWT. MongoDB only communicates with your backend using the database connection you've already established.

In one sentence
MongoDB authentication: "Can this backend connect to the database?"
JWT authentication: "Has this user already logged into the application?"

So they operate at different layers:

User
   │
   │ JWT
   ▼
Backend
   │
   │ MongoDB credentials
   ▼
MongoDB
